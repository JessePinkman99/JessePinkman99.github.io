如何搭建hexo博客并同步到GitHub仓库(Windows)

发表于 2020-04-22

为什么要搭建个人博客？

  • 记录知识点或者学习过程中遇到的坑，下次遇到可以看博客；

  • 部署到GitHub后可以让其他人看见，达到知识共享的目的，给自己一种成就感，并且可以保证自己的博客不会因为电脑的原因而丢失；

  • 个人博客可以自己定义风格，比在平台上搭建的博客更随心所欲；

  • 最重要的一点：用来装bei

什么是hexo？

  • hexo是一种基于nodejs的静态博客框架，它的风格简约大方，受到很多人的青睐，因此在搭建博客或者之后写博客的过程中遇到问题，很快就能在网上找到解决方法。另外，虽说基于nodejs，但搭建的过程并不需要任何一句nodejs的语法，搭建过程也很简单。

如何搭建hexo博客？

1.访问www.nodejs.org,下载nodejs,选择LTS长期支持版本；

2.按win+r，进入cmd命令行界面，检测nodejs和npm是否已经正确安装,npm是一种包管理器，会随着nodejs一起下载到本地；

node -v

npm -v

3.下载cnpm,因为用npm下载包非常的慢，所以我们先下载npm在国内的镜像cnpm,然后验证其版本号；

npm install -g cnpm --registry=https://registry.npm.taobao.org

cnpm -v

4.利用cnpm下载hexo框架；

cnpm install -g hexo-cli

5.创建你的博客文件夹，你可以直接用命令行在当前目录创建(一般是在C盘），然后cd命令进入文件夹，或者，通过鼠标右键在你想要的位置创建，然后cd命令进入文件夹的位置；
  • mkdir testblog cd testblog mkdir为创建文件夹命令，testblog为文件夹名字；
  • 自定义位置，以在桌面为例：cd C:\Users\Administrator\Desktop\testblog;
6.初始化博客；

hexo init

7.清理缓存，生成静态文件；

hexo c

hexo g

8.启动本地服务器；

hexo s

9.起飞！！！！！

​ 点这里跟着马老师起飞!!!

怎么把博客同步到自己的GitHub仓库？

1.登录GitHub账号，新建一个仓库；

确保你的仓库名的格式为:GitHub的名字加github加io，如图所示；并且勾选上readme

2.获取你的仓库url(点击Clone or download，然后选择协议，点右边的复制标志)；

这里有两种url，一种是走https协议的，另一种是走ssh协议的，两种的安全性都有保障，但从操作的角度来说，ssh操作稍微复杂但一劳永逸，https操作简单但每次同步都要输入账号密码；

3.更改配置文件；
  • 进入你的blog文件夹，打开_config.yml文件，用记事本或者notepad等都可以，；

  • 拉到最下面，找到Deployment这一项；

  • 更改如下，repo的值即为刚刚复制的url地址；
    注意！！每一条的冒号后都必须有一个英文空格

  • 保存退出。
4.下载插件；

cnpm install --save hexo-deployer-git

5.尝试上传到GitHub发现出错；

错误原因：刚才我们通过url知道GitHub仓库是谁，但这个仓库并不认识我们，我们“无名无姓”，是不可以上传的！！

6.下载安装Git工具，然后找到安装目录或者在开始菜单打开git-bash.exe；

https://git-scm.com/downloads




7.创建用户和邮箱；

这里的用户名和邮箱最好和GitHub相同，方便记忆，不同也没什么关系
gitbash里面是不能用ctrl+c和ctrl+v的，直接右键复制粘贴

git config --global user.name "yourname"
git config --global user.email "yourname@163.com"

对于前面选择ssh协议的朋友来说，还有操作（https的请下一步）

  • 创建SSH秘钥

ssh-keygen -t rsa -C "yourname@163.com"

这里后面的邮箱就是刚才注册的邮箱

  • 复制SSH秘钥
    进入到根目录，一般是C:\Users\Administrator,上图中有，打开id_rsa.pub文件，复制秘钥

  • 回到GitHub界面，点击右边小头像，选择settings;

  • 点击SSH and GPG keys

  • 粘贴秘钥

搞定！

8.上传博客到GitHub；

回到命令行界面，不是gitbash界面！
hexo c
hexo g
hexo d

对于选择https的朋友来说，他会提示你输入账号密码，即GitHub的账号密码，依次输入，回车，注意密码的输入是隐式的！
对于选择ssh的朋友来说，就直接成功了！

9.回到GitHub，刷新界面，会发现仓库里已经和博客文件夹同步了！

如何写博客呢？

用命令行输入hexo n "blogname"指令创建一片新的博客，然后到…blog\source\_posts文件夹下打开刚刚创建的blogname.md文件，需要用支持Markdown格式的编辑器打开(Visual Code、Typora等），写好博客(不知道Markdown语法的朋友自行百度，很简单)，保存退出，回到命令行界面，依次执行hexo c,hexo g,hexo s，再访问localhost:4000就能看到写好的博客已经发上传了，如果要同步到GitHub，hexo d。
